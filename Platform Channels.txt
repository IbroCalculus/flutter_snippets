
=============== What is Platform Channels? ===============
Platform Channels allow you to communicate between Flutter and Android/iOS native code.
A Platform Channel is the official mechanism Flutter uses to allow your Dart code to communicate with native code (Kotlin/Java on Android, Swift/Objective-C on iOS, C++ on Windows, etc.).

Because Flutter apps are written in Dart and compiled directly to machine code, Dart itself cannot directly call native device APIs (like battery levels, bluetooth, sensors, or secure storage). Platform channels act as the bridge or tunnel that passes messages back and forth between the Flutter side and the Native side.

--------- How it Works: The Message Bridge -------------
Think of a Platform Channel like a two-way radio between two different languages.
A message is sent from Flutter to the Native side, and then a message is sent back from the Native side to Flutter.

+----------------------+                      +------------------------+
|                      |   Platform Channel   |                        |
|  Flutter (Dart) API  | <==================> |  Native API (Host)     |
|                      |   (Async Messages)   |  (Kotlin/Swift/C++)    |
+----------------------+                      +------------------------+

1. The Request (Dart): Your Flutter code sends a message down a specific channel name (e.g., samples.flutter.dev/battery).
2. The Serialization: Flutter automatically translates the Dart data types (like Strings, Maps, or Ints) into standard binary formats.
3. The Execution (Native): The native side receives the message, unpacks it, runs the native platform-specific code (e.g., calling Android's BatteryManager), and gets the result.
4. The Response (Native -> Dart): The native side sends the result back up the channel, where it is decoded back into a Dart object.

Because this communication passes through an asynchronous bridge, all platform channel calls in Dart return a Future, meaning they must be awaited.


------------------ The Three Types of Platform Channels --------------
1. Method Channel: Used to call methods on a platform plugin.
2. Event Channel: Used to listen to events from a platform plugin.
3. Basic Message Channel: Used to send messages to a platform plugin.


1. MethodChannel (Most Common)
Used for one-off method calls and simple requests. Dart calls a native function by name and waits for a single response.
    Example use case: Requesting the current battery level, opening a native system dialog, or checking system storage space.

2. EventChannel
Used for continuous data streams. The native side sets up a listener and continuously streams data back to Dart over time.
    Example use case: Streaming live data from the phone's accelerometer/gyroscope, tracking GPS location changes, or listening to network connectivity status.

3. BasicMessageChannel
Used for passing continuous, asynchronous string or binary messages back and forth. It is less common but highly customizable for custom protocol buffers or raw byte streams.
