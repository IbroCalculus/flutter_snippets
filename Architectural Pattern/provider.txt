
========================= WHAT IS A PROVIDER? =========================
A Provider (or Riverpod/Bloc equivalent) is responsible for Managing application state and connecting UI to the domain/data layers.

A Provider is the layer that:
    holds data for the UI
    triggers loading states
    calls repositories/use cases
    notifies the UI when data changes

---------------- What Problem Does Provider Solve? -------------------
Without it, your UI would need to:
    - call repository directly
    - store data manually
    - track loading/error states
    - rebuild UI manually

That becomes messy very fast. Provider centralizes this logic.


---------------- Typical Role of Provider -------------------
A Provider usually:

1. Stores state
        loading: true/false
        data: List<Book>
        error: message

2. Calls Repository
        repository.getBooks();

3. Notifies UI
        notifyListeners();


------------------ Example (Conceptual) --------------------

class BookProvider extends ChangeNotifier {
  final BookRepository repository;

  BookProvider(this.repository);

  List<BookModel> books = [];
  bool isLoading = false;

  Future<void> fetchBooks() async {
    isLoading = true;
    notifyListeners();

    books = await repository.getBooks();

    isLoading = false;
    notifyListeners();
  }
}
