=============== FILES/FOLDERS ORGANIZATION vs ARCHITECTURAL PATTERNS ===============

Organizing files and folders is technically called Project Organization or Directory Structure or Project Structure or Folder Architecture
An Architectural Pattern (like MVC, MVVM, BLoC, or Clean Architecture) refers to how you separate your logic, data, and UI responsibilities in the code.
However, the two are deeply intertwined. Your folder structure is almost always a direct visual representation of your chosen architectural pattern.


=============== What is an architectural pattern? ===============

An architectural pattern is a standard way of organizing code in an application.

It defines:
    how different parts of the app are separated
    how those parts communicate
    where specific logic should live

Think of it as a blueprint for structuring software.

=================== Common Architectural Patterns (Code design patterns) ==============


    Pattern	                        Meaning
    MVC	                            Model View Controller
    MVVM	                        Model View ViewModel
    MVP	                            Model View Presenter
    Clean Architecture	            Layered architecture emphasizing independence
    BLoC Architecture	            Business Logic Component
    Redux	                        Centralized immutable state
    Flux	                        Unidirectional data flow
